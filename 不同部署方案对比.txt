对于基于 Docker 的外部演示，我来分析一下不同方案的优劣：

方案对比
1. ngrok（当前方案）
优点：

✅ 零配置，即开即用
✅ 自动 HTTPS
✅ 稳定的隧道连接
✅ 支持多种协议（HTTP/TCP）
✅ 有 Web 界面查看请求日志
缺点：

❌ 免费版有访问限制（每分钟 40 次请求）
❌ 免费版 URL 会变化（每次重启）
❌ 有警告页面（需点击继续）
❌ 依赖第三方服务
适用场景： 快速演示、临时分享、开发测试

2. Cloudflare Tunnel（推荐替代方案）
优点：

✅ 完全免费，无流量限制
✅ 可以绑定自定义域名
✅ 自动 HTTPS + CDN 加速
✅ 无访问警告页面
✅ 更稳定，Cloudflare 基础设施
✅ 支持访问控制和防火墙规则
缺点：

❌ 需要注册 Cloudflare 账号
❌ 需要拥有域名（可用免费域名）
实现方式：

bash
# 安装 cloudflared
brew install cloudflare/cloudflare/cloudflared  # macOS
# 或 https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/install-and-setup/installation/

# 登录
cloudflared tunnel login

# 创建隧道
cloudflared tunnel create credit-mgmt

# 启动隧道
cloudflared tunnel --url http://localhost:80
3. Tailscale Funnel（适合团队内部）
优点：

✅ 完全免费
✅ 基于 WireGuard，安全性高
✅ 可以限制访问权限
✅ 适合团队协作
缺点：

❌ 需要访问者也安装 Tailscale（或使用 Funnel 公开）
❌ 配置相对复杂
4. 自建服务器 + Nginx 反向代理
优点：

✅ 完全控制
✅ 无第三方依赖
✅ 可以绑定自己的域名
缺点：

❌ 需要购买服务器和域名
❌ 需要配置 SSL 证书
❌ 维护成本高
5. 云平台部署（生产级方案）
Railway / Render / Fly.io
优点：

✅ 免费额度足够演示使用
✅ 自动 HTTPS
✅ 支持 Docker 部署
✅ 自带域名
✅ CI/CD 集成
缺点：

❌ 免费版有资源限制
❌ 冷启动可能较慢
